Project Overview

This project demonstrates a production-ready AWS architecture built around an EC2 Auto Scaling Group (ASG). The goal was to handle changing traffic automatically, maintain high availability across multiple Availability Zones, and apply real-world cloud security best practices at every layer.

Users connect via Route 53 DNS and HTTPS (ACM), traffic is distributed by an Application Load Balancer, and EC2 instances run inside private subnets. Access is managed through IAM least-privilege roles and SSM Session Manager — no SSH ports exposed. The ASG was configured with a minimum of 1 and a maximum of 2 EC2 instances, ensuring the app stays online at all times while capping costs during scaling events.

Project Outline

"In this project, I built a secure and scalable AWS web architecture centered around an EC2 Auto Scaling Group. My objective was to handle changing traffic automatically, maintain high availability, and apply real-world cloud security best practices. The ALB distributes incoming requests across EC2 instances in private subnets, CloudWatch monitors CPU metrics and triggers alarms to scale, and all access is controlled via custom IAM roles with strict least privilege. SSM Session Manager eliminates the need for open SSH ports entirely."

Key Learnings
  • Scalability & Resilience: Auto Scaling is not just about performance — it is a resilience mechanism that keeps services alive during failures.
  • Defense-in-Depth: Security must be designed at every layer: HTTPS enforcement, private subnets, security groups, IAM, and no exposed SSH ports.
  • Least Privilege IAM: Writing a scoped IAM policy from scratch reinforced why broad permissions are dangerous in production environments.
  • CLI Automation: Using the AWS CLI for setup made the workflow faster, consistent, and reproducible — closer to how cloud engineers work professionally.
