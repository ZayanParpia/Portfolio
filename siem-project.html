Project Overview

This project is a hands-on cybersecurity home lab focused on Security Information and Event Management (SIEM), endpoint monitoring, log analysis, attack simulations, and SOC analyst workflows.

It is designed to simulate real-world SOC operations while building practical blue team experience. The core of this environment involves setting up Wazuh SIEM and deploying agents to collect extensive endpoint telemetry using tools like Sysmon and Auditd.

Project Outline

"In this project, I built a hands-on cybersecurity home lab focused on Security Information and Event Management (SIEM), endpoint telemetry, and SOC security workflows. The core of this environment involves setting up Wazuh SIEM as our centralized manager and deploying agents to collect host logs. By installing Sysmon for Linux and auditd on an Ubuntu endpoint, I configured detailed telemetry to capture process creations, system changes, and file activity, ensuring extensive visibility for security monitoring and threat hunting."

Simulation Architecture & Attack Flow

Network topology showing the Ubuntu target endpoint (Endpoint II) monitored by Wazuh Manager, Sysmon, and Auditd with escalation paths simulated directly on the host.

Telemetry & Alert Evidence

Visual evidence of each attack phase — endpoint status, terminal execution, and corresponding Wazuh rule triggers.

Endpoint II Active in Wazuh

Confirms the Ubuntu CLI target is online, registered, and forwarding telemetry to the SIEM manager prior to simulation.

Failed Sudo Password Guessing (Phase 1)

Terminal session showing repeated failed sudo authentication attempts, triggering the brute-force correlation rule.

Wazuh Rule 5557 — Failed Authentication Alert

Wazuh parses the auth log and fires rule 5557 (Level 5) per failed sudo. Custom rule 100002 aggregates 3 failures within 5 minutes into a high-severity brute-force alert.

Wazuh Rule 5403 — Successful Sudo Session (Phase 2)

Alert triggered when valid credentials are used for elevation. Wazuh logs the session, command, and user context.

Wazuh FIM Alert — Sudoers File Modified (Phase 3)

FIM detects an edit to /etc/sudoers, generating an immediate alert with file hash and modification timestamp.

Wazuh Alert — User Added to Sudo Group (Phase 4)

Auditd syscall log captures the usermod call adding a test account to the sudo group, confirming persistent privileged access.

Wazuh Rule 510 — Host Security Assessment

System integrity audit alert summarising host-level configuration checks during the simulation window.
