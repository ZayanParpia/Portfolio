Project Overview

The AWS Bastion Lab is a secure, cloud-native infrastructure project designed to simulate and implement a zero-trust access model. The core objective was to create a network where servers are hosted in strictly private subnets, inaccessible from the internet, and can only be managed via a hardened Bastion host (Jump Box).

Key to this project was the implementation of IAM-enforced sign-ins and AWS Systems Manager (SSM). This setup eliminates the need for persistent SSH keys or open port 22 on the private servers, significantly reducing the attack surface.

  • Segmented Architecture

    Logical separation of public entry points and private backend resources using VPC subnets.

  • Zero-Trust Access

    Enforced IAM policies and SSM for secure session management without open SSH ports.

  • Elastic Connectivity

    Configured Elastic IPs and NAT Gateways to allow private servers to update while remaining hidden.

  • Deep Monitoring

    Full visibility into network traffic and user actions via CloudTrail and VPC Flow Logs.
