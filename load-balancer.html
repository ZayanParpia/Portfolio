ALB Architecture

High-availability design featuring an Application Load Balancer distributing traffic across EC2 instances in multiple AZs.

Project Overview

The Application Load Balancer (ALB) project demonstrates a highly available and scalable web architecture on AWS. The primary goal was to ensure that a web application remains accessible even if individual server instances fail or experience high traffic.

By leveraging an Application Load Balancer, traffic is intelligently distributed across a fleet of EC2 instances. Combined with Auto Scaling, the system can automatically add or remove capacity based on real-time demand, optimizing both performance and cost.

  • High Availability

    Multi-AZ deployment ensuring service continuity during infrastructure failures.

  • Scaling Dynamics

    Auto Scaling groups reacting to CPU/Traffic metrics to maintain application responsiveness.

  • Health Monitoring

    Configured ALB health checks to automatically route traffic away from unhealthy instances.

  • Security Integration

    Integrated with Security Groups and SSL/TLS termination at the load balancer level.
