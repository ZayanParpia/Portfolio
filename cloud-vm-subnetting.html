Project Overview

This project was a deep dive into AWS networking fundamentals. I built a custom VPC from scratch, creating a realistic environment to understand how traffic flows between public and private subnets. I launched two Ubuntu EC2 instances in the same subnet and configured their Security Groups to allow specific traffic from each other only, verifying connectivity through ping and traceroute commands. A key component was security and monitoring: I enabled VPC Flow Logs to capture traffic data, storing these logs in CloudWatch encrypted with a custom KMS key. This allowed me to visualize accepted and rejected traffic events, directly observing the impact of my security group rules.

  • VPC Architecture

    Custom VPC design with granular subnet allocation for isolated workloads.

  • Security Groups

    Stateful filtering with least-privilege rules (SSH from specific IP, limited ICMP).

  • VPC Flow Logs

    Enabled comprehensive traffic monitoring for auditing and troubleshooting.

  • KMS Encryption

    Encrypted CloudWatch Log Groups using customer-managed keys (CMKs).

Process & Key Learnings

Project Goal: Build a small, realistic AWS network to learn how subnets, routing, ARP, security groups, and flow logs work.

What I Built: Two Ubuntu EC2 instances in different subnets within a custom VPC. Configured security groups, tested connectivity (ping/traceroute), and encrypted VPC Flow Logs using KMS.

Key Steps Completed:
  • Designed VPC and subnets (172.31.0.0/20).
  • Launched EC2 instances and verified private IPs.
  • Configured granular Security Groups for SSH/ICMP.
  • Assigned Elastic IPs for necessary public access.
  • Enabled VPC Flow Logs sent to CloudWatch.
  • Secured logs with a custom KMS key policy.
Security Practices Implemented

Implemented least-privilege Security Groups allowing strictly SSH and ICMP. Enabled VPC Flow Logs for full traffic visibility. Used AWS KMS to encrypt sensitive log data at rest, ensuring compliance and data security.

What I Learned
  • Networking Fundamentals: Deepened understanding of how private IPs, public IPs, subnets, and CIDR blocks interact within a VPC.
  • Routing & Connectivity: Learned how route tables control traffic and how ARP works for local reachability.
  • Visibility is Key: Discovered the value of VPC Flow Logs for troubleshooting connectivity issues and verifying dropped traffic.
  • Secure Encryption: Practiced creating and managing KMS keys to secure service logs.
